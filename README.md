## 📖 使用须知

**LLM 增强的研发提效平台** 是一个快速迭代的项目，旨在通过集成 **GitLab API**、**OpenAPI** 和 **大规模语言模型（LLM）** 提升开发效率。该平台为开发者提供了一系列的自动化工具，包括自动文档解析、代码变更跟踪、以及基于 LLM 的项目知识问答功能。

遇到问题？欢迎在 [Issues](https://github.com/Thomaswoodd-code/llm-enhanced-dev-platform/issues) 提交反馈，我们会尽快处理！更多功能正在开发中，敬请期待！✨

## ✨ 功能一览

- **GitLab 仓库接入**：一键拉取 **GitLab** 项目中的仓库信息、分支、提交记录、合并请求（MR）、流水线状态（CI/CD）。
- **项目文档解析**：自动解析 **README**、接口文档和 **OpenAPI** 文件，生成清晰易懂的项目文档，帮助开发者快速理解项目内容和接口功能。
- **代码变更摘要**：自动生成提交信息摘要和代码变更分析，根据 `commit` 或 `diff` 自动生成变更说明，快速了解本次更新的内容。
- **API 变更影响分析**：根据接口文档和提交记录分析，自动评估 API 变更对项目其他部分的影响，确保项目稳定性。
- **项目问答助手**：基于项目文档和代码注释，使用 **LLM** 提供基于上下文的问答功能，自动回答如 “项目如何启动？”、“某接口功能是什么？” 等问题。
- **CI/CD 流水线状态**：查看并展示 GitLab 上的最新 CI/CD 流水线状态，包括构建、测试、部署结果，确保开发者了解最新的构建状态。
- **配置管理与版本控制**：自动追踪和管理项目中的配置文件（如 YAML、JSON），并记录配置文件版本与提交记录的映射，确保配置的可追溯性。
- **智能文档生成**：自动生成和更新项目文档、API 接口文档、用户手册等，减少人工编写文档的工作量。
- **代码质量分析**：自动生成项目的代码质量报告，帮助开发者了解代码中的潜在问题或改进空间。
- **多语言支持**：支持多个语言的文档解析和生成，帮助开发者跨语言环境进行协作。
