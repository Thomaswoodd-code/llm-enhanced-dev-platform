# 开发手册

## 项目简介

**LLM Enhanced Dev Platform** 是一个集成 **GitLab API**、**OpenAPI** 和 **大规模语言模型（LLM）** 的研发提效平台，旨在提升开发效率并简化常见开发任务。该平台提供了自动化工具来帮助开发者：

\- 自动解析项目文档
\- 生成代码变更摘要
\- 提供基于 LLM 的项目问答助手
\- 实时查看 **GitLab** CI/CD 流水线状态
\- 智能分析 API 变更对项目的影响

通过将这些功能自动化，开发者可以专注于核心业务代码，而不必重复做那些繁琐的操作。

## 文件夹结构与作用

### 1. **`frontend/` - 前端文件夹**

\- **作用**：负责项目的用户界面展示和交互。
\- **内容**：
  \- **`index.html`**：主页面文件，展示项目相关的内容。
  \- **`assets/`**：存放静态资源（如图片、样式表、字体等）。
  \- **`scripts/`**：存放前端 JavaScript 文件，包含项目的前端逻辑。

### 2. **`backend/` - 后端文件夹**

\- **作用**：处理前端请求、与 GitLab API 和 OpenAPI 交互，并管理业务逻辑。
\- **内容**：
  \- **`src/`**：包含后端的核心代码，主要由以下几个部分构成：
    \- **`controllers/`**：处理前端请求的路由和控制器。
    \- **`models/`**：定义与数据库交互的模型。
    \- **`services/`**：处理项目的业务逻辑。
    \- **`app.js`**：后端的主应用文件，设置 API 路由和 Express 服务。
  \- **`.env`**：存储环境变量（如数据库连接信息等）。
  \- **`package.json`**：管理后端项目依赖。

### 3. **`database/` - 数据库文件夹**

\- **作用**：管理数据库模型和相关的迁移文件。
\- **内容**：
  \- **`migrations/`**：数据库迁移文件，用于更新数据库结构。
  \- **`models/`**：定义数据库模型，帮助与数据库交互。
  \- **`schema.sql`**：数据库初始化文件，定义数据库的表结构。
